Setting Up My WSL Environment

Oct 24, 2017 | 2 minutes read
Share this: Twitter Facebook Pinterest Google+

Microsoft’s Windows Fall Creator Update was the end of Windows Subsystem for Linux (WSL)’s beta phase. As a full feature of Windows, I wanted to integrate it fully into my work environment. The primary objective for this task was to make a functional development environment with little need for leaving WSL. I chose Ubuntu from the Windows Store for my Linux distro.

One of the first things I did was install an X11 server so I can use GUIs for the programs installed in WSL: 1. Install vcxsrv 2. Launch it from the Start menu and leave it running

Then, I had to set up WSL and Ubuntu by doing the following: 1. Enable WSL from “Turn Windows Features on or off” 2. Download Ubuntu from Windows Store 3. Run it for the first time. Enter username and password

As is, Ubuntu will only be able to open programs in the terminal and not as a GUI. To do that, I had to do the following steps: 1. nano ~/.bashrc 2. eWrite xport 0:0 at the end of the file

That is all for the basic installation of Ubuntu and WSL. These are additional steps I took to set up my personal development environment. It will install Emacs (with my .emacs.d), Git, GCC, and TexLive. 1. sudo add-apt-repository ppa:kelleyk/emacs 2. sudo apt-get update 3. sudo apt-get install git gcc emacs texlive-full 4. git clone https://github.com/omn0mn0m/.emacs.d.git --recursive

I hope that this blog post will help with getting up and running with your own WSL environment. For a list of Linux packages and their compatibility with WSL, you can check out this repo.

NOTE: I was not able to get a full desktop environment working. When I figure that out, I will post it here.
